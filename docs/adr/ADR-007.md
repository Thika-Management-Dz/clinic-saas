# ADR-007: FDI Two-Digit Tooth Notation Only (ISO 3950:2016)

- **Status:** Accepted
- **Date:** 2026-07-06
- **Blueprint references:** §9.4.1, §9.4.2
- **Supersedes:** none
- **Superseded by:** none

## Context

Algeria inherited the **French dental education system**, which uses **FDI
Two-Digit Notation** (Fédération Dentaire Internationale). FDI is the world's
most commonly used dental notation and is **ISO 3950:2016** (the 1984
edition was withdrawn and replaced by the 2009 and 2016 editions).

Dental notation alternatives include:

- **Universal** (US): single number 1–32 for permanent, A–T for primary.
  Used in the United States; unfamiliar to Algerian-trained dentists.
- **Palmer**: quadrant symbol + tooth number (e.g., `┘8` for upper-right
  3rd molar). Historic, mostly superseded by FDI.

The clinic SaaS serves Algerian clinicians; the storage format must match
their training, but US-trained dentists (rare) need a display-only conversion.

## Decision

Adopt **FDI Two-Digit Notation (ISO 3950:2016)** as the **only storage
format**. Display-only conversion to Universal (US) notation is optional and
cosmetic; no other notation is stored.

### Storage

- **Column type**: `INT` (2-digit integer). Column name: `tooth_fdi`
  (see `docs/conventions/naming.md`).
- **Validation set** (enforced at the Drizzle schema level via a CHECK
  constraint and a Zod schema):

  | Dentition | Quadrants | Tooth range | Valid set |
  |---|---|---|---|
  | Permanent (32 teeth, 8 per quadrant) | 1, 2, 3, 4 | 1–8 | `11..18, 21..28, 31..38, 41..48` |
  | Primary (20 teeth, 5 per quadrant) | 5, 6, 7, 8 | 1–5 | `51..55, 61..65, 71..75, 81..85` |

- **Quadrant semantics**:
  - `1` upper right permanent, `2` upper left permanent,
  - `3` lower left permanent, `4` lower right permanent.
  - `5` upper right primary, `6` upper left primary,
  - `7` lower left primary, `8` lower right primary.
- **Tooth number semantics** (per quadrant): `1` = central incisor → `8` =
  3rd molar. So `11` = upper-right central incisor, `48` = lower-right 3rd
  molar.

### Validation

```ts
// packages/contracts/src/dental/fdi.ts
const PERMANENT_QUADRANTS = [1, 2, 3, 4] as const;
const PRIMARY_QUADRANTS   = [5, 6, 7, 8] as const;
const PERMANENT_TOOTH_RANGE = [1, 2, 3, 4, 5, 6, 7, 8] as const;
const PRIMARY_TOOTH_RANGE   = [1, 2, 3, 4, 5] as const;

export const validFdi = (n: number): boolean => {
  const q = Math.floor(n / 10);
  const t = n % 10;
  if (PERMANENT_QUADRANTS.includes(q as 1|2|3|4)) return PERMANENT_TOOTH_RANGE.includes(t as 1|2|3|4|5|6|7|8);
  if (PRIMARY_QUADRANTS.includes(q as 5|6|7|8))   return PRIMARY_TOOTH_RANGE.includes(t as 1|2|3|4|5);
  return false;
};
```

### Display-only conversion to Universal (US)

Optional, behind a per-user preference. Conversion is **lossy in the
directional sense** (Universal numbers do not encode quadrant) and is used
only for US-trained dentists who prefer it on screen. **Stored value is
always FDI**; UI display layer applies the conversion function.

### Tooth surfaces — bitfield INT

Companion decision (§9.4.2): tooth surfaces are stored as a **bitfield INT**
on each `procedure` or `tooth_finding` row.

| Surface | Bit | Value |
|---|---|---|
| Mesial (M) | 0 | 1 |
| Occlusal (O) | 2 |
| Distal (D) | 4 |
| Buccal (B) | 8 |
| Lingual (L) | 16 |
| Incisal (I) | 32 |
| Palatal (P) | 64 |

A multi-surface restoration (e.g., MOD amalgam) = `1 + 2 + 4 = 7` in a single
integer column.

## Consequences

**Positive**

- Matches Algerian clinician training — no cognitive translation at the chair.
- 2-digit INT is the smallest possible storage (vs strings like `"UR8"` or
  `"48"`); index-friendly, comparison-friendly.
- Validation set is a closed enum at the DB layer — bad data cannot land.
- Surfaces as bitfield INT = one column, no join table, multi-surface
  restorations in O(1).

**Negative**

- US-trained dentists need display-only conversion; mitigation is a per-user
  preference toggle.
- Bitfield INT is less readable than `['M','O','D']` string array; mitigated
  by helper functions `decodeSurfaces(bitfield: number): Surface[]` and
  `encodeSurfaces(surfaces: Surface[]): number` in `packages/contracts`.

## Alternatives Considered

### Universal (US) notation

Single number 1–32 for permanent teeth. Rejected: unfamiliar to
Algerian-trained dentists; would require translation in the opposite
direction at every chair. Available only as an optional display conversion.

### Palmer notation

Quadrant symbol + tooth number (e.g., `┘8`). Historic; largely superseded
by FDI internationally and not used in Algerian dental education. Rejected.

### Store tooth number as a string (e.g., `"11"`, `"48"`)

Same information content as INT but larger storage, slower comparison, and
no CHECK-constraint enum. Rejected; INT is strictly better.
